**Q1. 상태코드 301 (Permanently Moved)과 302 (Found)의 차이를 자세하게 이해하기가 힘듭니다.**
<br/>
HTTP 상태 코드 301(Moved Permanently) 과 302(Found) 는 둘 다 리다이렉트를 나타내지만,
클라이언트가 이 리다이렉트 정보를 얼마나 오래 믿어도 되는지에 대한 의미가 다르다.
- ```301 (Moved Permanently)```
리소스의 위치가 영구적으로 변경되었을 때 사용한다.
브라우저나 중간 캐시들은 이 정보를 비교적 오래 저장할 수 있고,
이후 같은 URL로 요청이 오면 굳이 서버까지 가지 않고
캐시에 저장된 리다이렉트 정보를 사용해 바로 새 URL로 요청을 보낼 수 있다.

- 요청이 매우 많고,
단축 URL이 가리키는 원본 URL이 거의 바뀌지 않는다면
→ 301을 써서 캐시와 브라우저가 리다이렉트를 기억하게 만드는 것이 성능 측면에서 유리하다.

- ```302 (Found)```
리소스가 일시적으로 다른 위치로 이동했을 때 사용한다.
나중에 다시 원래 URL이 유효해질 수 있다고 가정하기 때문에,
브라우저는 보통 301만큼 강하게 캐싱하지 않고
재방문 시 다시 서버에 요청을 보내 최신 리다이렉트 상태를 확인하는 경우가 많다.

- 반대로, 단축 URL이 가리키는 원본 URL을 나중에 바꿀 가능성이 크거나,
항상 서버를 들르게 하고 싶다면
→ 302처럼 “일시적 리다이렉트”를 사용해서
브라우저가 매번 서버를 거쳐 가도록 하는 편이 좋다.

**Q2. NTP(Network Time Protocol)에 대해?**
<br/>
= 컴퓨터나 서버의 시간을 네트워크를 통해 자동으로 정확하게 맞추는 프로토콜
전 세계에 있는 표준 시계와 동기화된 시간 서버에서 시간을 받아서 자신의 시스템 시간을 조정
동작 방식 : 중앙 시간 서버를 두고 모든 장비가 그 서버와 동기화

a. 클라이언트(서버)가 NTP server에 요청을 보냄
b. 시간 서버는 현재 정확한 시간을 반환
c. 클라이언트는 네트워크 latency를 계산해서 자신의 시스템 시간을 조금씩 보정
d. 정기적으로 서버와 재동기화

중앙 시간 서버를 두고 모든 장비가 그 서버와 동기화 하는 방식 => 전체 시스템의 시간이 항상 동일하게 유지됨
